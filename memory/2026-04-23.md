## 🚀 jorge-ai-demo 项目开发部署 (00:22 - 01:07)

### 需求
- 前后端分离: Java Spring Boot 2 + Vue 3
- 管理平台: 账号登录、用户管理、登录记录
- 数据库: jorge-demo
- Admin账号管理能力

### 后端开发 (13个Java文件, 739行代码)
- **JWT认证**: JwtUtil + SecurityConfig
- **用户系统**: User实体 + UserService + AuthService
- **登录日志**: LoginLog实体自动记录IP/UA/时间
- **权限控制**: @PreAuthorize("hasRole('ADMIN')")
- **API接口**: /api/auth/register, /api/auth/login, /api/users/**

### 前端开发 (Vue 3 + Element Plus)
- 登录页 (紫蓝渐变品牌感)
- 后台布局 (侧边栏+顶部导航)
- 仪表盘、用户管理、登录记录、个人中心
- 自动JWT拦截器

### 遇到的坑
1. **OOM Kill** - 4G内存不够Maven编译，加2G Swap + 限制内存参数
2. **Java版本** - 服务器只有Java 8，子Agent用了Java 9+的`Map.of()`和`record`
3. **Spring Boot版本** - 子Agent写了Spring Boot 3代码，服务器环境只能用Spring Boot 2
4. **JJWT API** - 0.9.1和0.12.3 API不兼容
5. **SecurityConfig** - Spring Security 5和6配置方式不同

### 修复方案
- 手动修复所有Java文件，替换jakarta为javax
- 重写AuthController移除record类型
- 重写JwtUtil使用JJWT 0.9.1 API
- 重写SecurityConfig使用WebSecurityConfigurerAdapter
- 限制Maven内存: `-Xmx256m -XX:MaxMetaspaceSize=128m`

### 部署结果
- ✅ 后端启动: 7.1秒, 端口8080
- ✅ 前端编译: 13.24秒, dist/ 目录
- ✅ Nginx配置: 80端口代理/api到8080
- ✅ Admin账号: admin / admin123 (角色ADMIN)

## 🎯 经验总结

1. **4G内存真的不够** - Java项目编译很容易被OOM kill，必须加swap + 限制内存
2. **子Agent代码要审查** - 生成的代码可能用新版本API，和服务器环境不匹配
3. **SSH密钥比密码安全** - 永远不要直接给用户密码
4. **Maven编译慢** - 首次下载依赖需要几分钟，后续增量编译快
